Image Filter Program – Implementation Plan and Architecture
============================================================

High-Level Architecture Overview
-------------------------------
Core Components:

- CommandLineParser: Parses and validates command-line arguments into structured filter specs
- FilterFactory: Creates filter objects (`IFilter`) from parsed specifications
- FilterPipeline: Stores filters and applies them in sequence
- IFileManager + BMPFileManager: Loads/saves BMP images to/from a `Picture` class
- IFilter (interface): Base class for all filters
- Picture + Pixel: In-memory representation of an image
- ConvolutionFilter: Shared logic for all convolution-based filters (blur, edge, etc.)
- Exception classes: Handle and report runtime and logic errors cleanly

Detailed Class Specifications
-----------------------------

CommandLineParser
-----------------
class CommandLineParser {
public:
    CommandLineParser(int argc, char* argv[]);
    std::string getInputFilename() const;
    std::string getOutputFilename() const;
    std::vector<FilterSpec> getFilterSpecs() const;
    bool isHelpRequested() const;
};

- Validates presence of -i and -o
- Parses filter options in order
- On error, throws InvalidParameterException

FilterFactory
-------------
class FilterFactory {
public:
    static std::unique_ptr<IFilter> create(const FilterSpec& spec);
};

- Uses spec.name to choose filter
- Parses spec.parameters for arguments
- Throws InvalidParameterException if parsing fails

FilterPipeline
--------------
class FilterPipeline {
public:
    void addFilter(std::unique_ptr<IFilter> filter);
    void apply(Picture& image) const;
private:
    std::vector<std::unique_ptr<IFilter>> filters;
};

- Applies all filters in order to the image

IFileManager / BMPFileManager
-----------------------------
class IFileManager {
public:
    virtual Picture load(const std::string& filename) = 0;
    virtual void save(const Picture& picture, const std::string& filename) = 0;
};

class BMPFileManager : public IFileManager {
public:
    Picture load(const std::string& filename) override;
    void save(const Picture& picture, const std::string& filename) override;
};

- Validates BMP format: 32-bit, uncompressed
- Throws FileOpenException or InvalidFormatException if invalid

Picture and Pixel
-----------------
class Pixel {
public:
    uint8_t r, g, b, a;  // alpha ignored for processing
};

class Picture {
public:
    int width, height;
    std::vector<Pixel> data;

    Pixel& at(int x, int y);
    const Pixel& at(int x, int y) const;
};

IFilter and Filter Subclasses
-----------------------------
class IFilter {
public:
    virtual void apply(Picture& img) const = 0;
    virtual ~IFilter() = default;
};

// Examples:
class BlurFilter : public ConvolutionFilter { ... };
class EdgeDetectFilter : public ConvolutionFilter { ... };
class GammaCorrectionFilter : public IFilter { ... };
class ContrastBrightnessFilter : public IFilter { ... };
class MedianFilter : public IFilter { ... };
class RGBCorrectionFilter : public IFilter { ... };

ConvolutionFilter
-----------------
- Abstract base for filters that use a kernel
- Shared logic for convolution (looping over neighbors, applying kernel)
- Used by BlurFilter, EdgeDetectFilter, etc.

Exception Handling
------------------
Custom Exception Classes:

- FileOpenException (std::runtime_error) – file cannot be opened or written
- InvalidFormatException (std::runtime_error) – file is not a 32-bit BMP
- InvalidParameterException (std::logic_error) – user gave invalid argument or filter parameter

Each stores a message and overrides what() from std::exception.

Command-Line Interface
----------------------
Usage:
  imgfilter -i <input.bmp> -o <output.bmp> [filters]

Available Filters:
  --contrast <factor>
  --brightness <value>
  --blur <radius>
  --median <radius>
  --edge
  --offset <r> <g> <b>
  --gamma <value>
  -h, --help

Example:
  imgfilter -i photo.bmp -o out.bmp --contrast 1.5 --blur 2 --gamma 2.2

Error Handling Rules
--------------------
- User is notified with meaningful messages.
- In case of error, the program allows retrying by asking for corrected inputs.

Functional Specification (per assignment)
-----------------------------------------
Készítsen C++ programot, amely képes 32bites Windows Bitmap 
formátumú képfájl betöltésére. Minden képpontot 32 bites mező 
kódol (unsigned int), tartalmazza a három színcsatorna intenzitását 
és az átlátszóságot bájtonként (4x8bit). Az átlátszósággal nem kell 
foglalkozni, minden esetben tekinthető 255 értékűnek. 
Implementáljon több szűrő algoritmust, mely a bemeneti képen dolgozik, 
az eredményt újabb képfájlba menti. Opcionálisan több szűrő összekötését 
is megvalósíthatja, ebben az esetben csak az utolsó szűrő eredménye íródik 
ki fájlba. Akár a betöltés és a mentés is tekinthető egy-egy szűrőnek.
Használjon objektumorientált reprezentációt és öröklést.
Készítsen tesztalkalmazást a funkciók kipróbálásához!

Javasolt képszűrők:
    • Kontraszt állítás
    • Életlenítés (blur, konvolúcióval)
    • Éldetektálás
    • Szín módosítás (csatornánként megadható +- ofszetek)
    • Gamma korrekció
    • Medián szűrő
